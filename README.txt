--Балансировка корпуса--

Установка дополнительных модулей не требуется.
В командной строке ввести:
python postediting.py <кол-во словоупотреблений> <год начала периода> <месяц начала периода> <год конца периода> <месяц конца периода> <домен>

Можно вводить несколько доменов через пробел.
Общее количество слов будет поделено поровну на каждый домен.

Пример:
python postediting.py 1000000 2011 01 2014 12 izvestia sovsport

Программа postediting.py должна лежать в той же папке, что и папки с выкаченными статьями.
Папки со статьями должны называться как домены газет. Далее в них должны лежать папки с годами и месяцами.
Примерная структура дерева папок:

postediting/
├── postediting.py
├── izvestia/
│   ├── 2015/
│   │    ├── 01/
│   │	 │    ├── 483784.xml
│   │	 ├── 02/
│   │	 ├── 03/
│   ├── 2014/
│   ├── 2013/
├── ria/
├── rbc/

На выходе внутри папки postediting/ создастся папка results/ с копиями статей.

--Преобразование xml файлов и создание мета-таблицы--

В командной строке ввести:
python postediting.py <путь к папке с cбалансированным корпусом> <путь к новой папке> <название_таблицы.csv> -convert

Пример:
python postediting.py postediting/results postediting/new_results meta.csv -convert

На выходе внутри папки postediting/ создастся папка new_results/.
В этой папке статьи будут перекодированы в windows-1251 и не будут содержать метаинформацию.
В корневой папке будет создан файл new_results.csv с метаинформацией по каждой статье.
